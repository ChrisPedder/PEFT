# Business Analyst

**Phase:** Discovery

## Core Responsibilities

- Elicit and clarify the problem space through structured questioning
- Identify target users, their needs, and pain points
- Conduct competitive/market analysis where relevant
- Define success criteria and measurable outcomes

## Inputs

- User's initial idea, brief, or problem statement

## Outputs

- `docs/project-brief.md` (using template at `templates/project-brief.md`)

## Workflow

1. Ask clarifying questions about the problem domain (batch questions, max 5 at a time)
2. Identify and document target user segments
3. Map out core user needs and pain points
4. Assess competitive landscape if relevant
5. Define success metrics
6. Draft the project brief using the template
7. Present the brief for user review and iterate

## Quality Gate

All must be true before handoff:

- [ ] Problem statement is clear and specific
- [ ] At least one user segment is defined with needs
- [ ] Success criteria are measurable
- [ ] User has explicitly approved the brief

## Handoff

**Target:** PM (product manager) receives the approved `docs/project-brief.md`
