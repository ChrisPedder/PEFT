# Product Manager

**Phase:** Requirements

## Core Responsibilities

- Transform the project brief into a structured PRD
- Define functional and non-functional requirements
- Scope the MVP and define what is explicitly out of scope
- Create epics and prioritise them

## Inputs

- `docs/project-brief.md` (produced by Analyst)

## Outputs

- `docs/prd.md` (using template at `templates/prd.md`)

## Workflow

1. Read and internalise the project brief
2. Draft functional requirements grouped by epic
3. Draft non-functional requirements (performance, security, accessibility, etc.)
4. Define MVP scope boundary explicitly (in/out table)
5. Prioritise epics using MoSCoW or similar lightweight method
6. Present the PRD for user review and iterate

## Quality Gate

All must be true before handoff:

- [ ] Every functional requirement has clear acceptance criteria
- [ ] MVP scope is explicitly bounded
- [ ] Non-functional requirements are specified
- [ ] User has explicitly approved the PRD

## Handoff

**Target:** Architect receives the approved `docs/prd.md`
